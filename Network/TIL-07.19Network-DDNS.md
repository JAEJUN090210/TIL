# TIL - 07.19 / Network - DDNS

## Summary

- **DDNS(Dynamic Domain Name System)는 변경되는 공인 IP 주소를 도메인 이름과 자동으로 연결하여 항상 동일한 도메인으로 접속할 수 있도록 하는 서비스**

------

## Body

- **DDNS (Dynamic Domain Name System)**
  - 인터넷 회선의 **변경되는 공인 IP 주소를 도메인에 자동으로 반영**하는 기술
  - 고정 IP 없이도 항상 같은 도메인으로 서버에 접속 가능
  - NAS, 홈 서버, CCTV, 개인 웹 서버 등에 많이 사용
- **주요 구성 요소**
  - **Public IP (공인 IP)**
    - 인터넷 서비스 제공업체(ISP)가 할당하는 외부 IP 주소
    - 대부분 가정용 인터넷은 일정 시간이 지나면 IP가 변경될 수 있음
  - **Domain (도메인)**
    - 사람이 기억하기 쉬운 주소
    - 예: `example.ddns.net`
  - **DDNS Client**
    - 현재 공인 IP를 주기적으로 확인
    - IP가 변경되면 DDNS 서버에 새로운 IP를 자동 등록
  - **DDNS Server**
    - 클라이언트로부터 전달받은 최신 IP 정보를 저장
    - DNS 레코드를 자동으로 갱신
  - **DNS**
    - 도메인을 실제 IP 주소로 변환하는 시스템
    - DDNS는 DNS를 자동으로 업데이트하는 방식으로 동작
- **일반적인 DDNS 동작 흐름**
  1. 사용자가 DDNS 도메인을 등록
  2. DDNS Client가 현재 공인 IP 확인
  3. 공인 IP가 변경되면 DDNS 서버에 변경된 IP 전송
  4. DDNS 서버가 DNS 레코드 자동 갱신
  5. 사용자는 항상 동일한 도메인으로 서버 접속
  6. DNS는 최신 공인 IP를 반환하여 연결 수행

------

## (+)

- 고정 IP 없이도 도메인으로 접속 가능
- 공인 IP 변경 시 자동으로 DNS 갱신
- 홈 서버, NAS, CCTV 운영에 적합
- 설정이 비교적 간단함

------

## (-)

- DNS 갱신까지 약간의 지연이 발생할 수 있음
- 포트 포워딩 또는 리버스 프록시 설정이 필요한 경우가 많음
- 무료 DDNS 서비스는 도메인 유지 조건이나 기능 제한이 있을 수 있음
- 고정 IP 환경보다 안정성이 다소 떨어질 수 있음
