# TIL - 06.29 / Network - Network Topology

## Summary

- **토폴로지(Topology)는 네트워크 장치들이 서로 연결되는 구조와 데이터가 흐르는 방식을 의미**

------

## Body

- **토폴로지**
  - 네트워크 장치(Node)의 연결 구조
  - **물리적 토폴로지**: 실제 배선 및 장치 연결 방식
  - **논리적 토폴로지**: 데이터가 이동하는 방식
- **Ring (링형)**
  - 장치를 원형으로 연결
  - 토큰(Token)을 이용해 순차적으로 데이터 전송
  - 장애 발생 시 네트워크 전체에 영향
- **Star (별형)**
  - 중앙 스위치(또는 허브)에 모든 장치 연결
  - 현재 가장 많이 사용하는 구조
  - 중앙 장비 장애 시 전체 네트워크 영향
- **Tree (트리형)**
  - 여러 개의 Star 구조를 계층적으로 연결
  - 확장성이 뛰어나 대규모 네트워크에 적합
- **Mesh (그물형)**
  - 여러 경로를 통해 통신 가능
  - 장애 발생 시 우회 가능
  - 높은 안정성과 높은 구축 비용
- **Bus (버스형)**
  - 하나의 통신 회선을 여러 장치가 공유
  - 구축 비용이 저렴하지만 병목과 충돌이 발생하기 쉬움
- **Linear (선형)**
  - 장치를 일렬로 연결
  - 구조는 단순하지만 중간 노드 장애 시 이후 구간 통신 불가
