# TIL - 07.07 / Network - Packet

## Summary

- **패킷(Packet)은 네트워크에서 데이터를 전송하기 위해 일정한 크기로 분할한 데이터의 최소 전송 단위이다.**

------

## Body

- **패킷(Packet)**
  - 데이터를 효율적으로 전송하기 위해 작은 단위로 나눈 데이터
  - 인터넷에서는 대부분 패킷 단위로 통신
  - 목적지에서 다시 조립되어 원래 데이터로 복원됨
- **패킷의 구성**
  - **Header**
    - 출발지 IP
    - 목적지 IP
    - 포트 번호
    - 프로토콜 정보
    - 순서 번호 등 제어 정보
  - **Payload**
    - 실제 전송되는 데이터
  - **Trailer**
    - 오류 검출 정보(CRC 등)
    - 데이터 무결성 확인
- **패킷 전송 과정**
  1. 데이터를 여러 개의 패킷으로 분할
  2. 각 패킷에 헤더 정보 추가
  3. 라우터를 거쳐 목적지로 전송
  4. 목적지에서 순서대로 재조립
- **패킷 스위칭(Packet Switching)**
  - 각 패킷이 독립적으로 최적의 경로를 선택하여 전송
  - 네트워크 상황에 따라 서로 다른 경로를 이용할 수 있음
  - 인터넷의 기본 데이터 전송 방식

------

## (+)

- 네트워크 자원을 효율적으로 사용
- 장애 발생 시 다른 경로로 우회 가능
- 여러 사용자가 동시에 네트워크 공유 가능
- 일부 패킷 손실 시 필요한 부분만 재전송 가능(TCP)

------

## (-)

- 패킷이 서로 다른 경로를 이용하면 순서가 달라질 수 있음
- 네트워크 혼잡 시 지연(Latency) 발생
- 패킷 손실(Packet Loss)이 발생할 수 있음
- 헤더 정보가 포함되어 오버헤드가 발생
