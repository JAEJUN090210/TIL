# TIL - 06.28 / IP & Subnet Mask

## Summary

- IP 주소는 **네트워크와 장치를 식별하는 주소**이며, 서브넷 마스크는 **Network ID와 Host ID를 구분하는 기준**

------

## Body

- **IP 주소**
  - 네트워크에서 장치를 식별하는 고유 주소
  - IPv4는 **32bit(4옥텟)** 구조
  - **Network ID + Host ID**로 구성
  - 주소 부족 문제로 **CIDR(Classless Inter-Domain Routing)** 방식 도입
- **서브넷 마스크**
  - IP 주소에서 **네트워크 영역과 호스트 영역을 구분**하는 32bit 값
  - `255.255.255.0` = `/24`처럼 표현
  - 컴퓨터는 **IP와 서브넷 마스크를 AND 연산**하여 네트워크 주소를 계산
- **CIDR**
  - 기존 A/B/C 클래스의 비효율을 해결하기 위해 등장
  - 필요한 만큼의 IP만 할당하여 주소 낭비 감소
  - 예) 300대 규모 → `/23` 대역 사용
- **예시 (/22)**
  - `192.168.0.x ~ 192.168.3.x` → 같은 네트워크
  - `192.168.4.x` → 다른 네트워크

------

## (+)

- IP를 효율적으로 사용할 수 있음
- 네트워크를 필요한 크기로 자유롭게 분할 가능
- 같은 네트워크 여부를 빠르게 판단할 수 있음
- 라우팅 효율 향상

------

## (-)

- 서브넷 계산이 익숙하지 않으면 이해하기 어려움
- 잘못 설정하면 통신 불가 또는 네트워크 충돌 발생
- 네트워크 규모에 맞지 않는 서브넷은 IP 낭비 또는 부족을 초래할 수 있음
