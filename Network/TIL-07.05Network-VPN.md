# TIL - 07.05 / Network - VPN

## Summary

- **VPN(Virtual Private Network)은 인터넷 상에 암호화된 터널을 생성하여 안전하게 데이터를 전송하는 기술이다.**
- **데이터의 기밀성과 무결성은 보호하지만, 익명성을 완전히 보장하는 기술은 아니다.**

------

## Body

### VPN이란?

- 공용 인터넷을 이용해 **사설 네트워크(Private Network)**처럼 안전하게 통신하는 기술
- 사용자와 VPN 서버 사이의 모든 트래픽을 암호화하여 외부에서 내용을 확인하기 어렵게 만든다.
- 사용자는 VPN 서버를 통해 인터넷에 접속하므로 외부에서는 VPN 서버의 IP 주소가 보인다.

### 동작 과정

1. 사용자가 VPN 서버에 연결
2. VPN 클라이언트와 서버가 암호화 세션 생성
3. 모든 네트워크 패킷을 암호화하여 터널로 전송
4. VPN 서버가 복호화 후 목적지 서버로 전달
5. 응답도 동일한 과정을 거쳐 사용자에게 전달

### 터널링(Tunneling)

- 기존 IP 패킷을 암호화한 뒤 새로운 패킷에 캡슐화(Encapsulation)하여 전송하는 기술
- 외부에서는 내부 데이터 내용을 확인할 수 없으며, VPN 서버에서 복호화되어 원래 목적지로 전달된다.

### 주요 VPN 프로토콜

- **WireGuard**
  - 최신 VPN 프로토콜
  - 높은 성능과 낮은 지연시간
  - 코드가 간결하여 보안성이 높음
- **OpenVPN**
  - SSL/TLS 기반 오픈소스 프로토콜
  - 안정성과 호환성이 뛰어나 가장 널리 사용됨
- **IPsec**
  - 네트워크 계층에서 동작
  - 기업 환경의 Site-to-Site VPN에서 많이 사용
- **L2TP/IPsec**
  - L2TP와 IPsec을 함께 사용하는 방식
  - 과거에는 많이 사용했지만 최근에는 WireGuard와 OpenVPN이 더 많이 활용됨

### VPN의 활용 사례

- 공용 Wi-Fi에서 안전한 통신
- 기업 내부망 원격 접속
- 해외에서 사내 시스템 접속
- 지역 제한 서비스 이용
- 인터넷 트래픽 암호화

------

## (+)

- 인터넷 트래픽 암호화
- 공용 Wi-Fi에서도 안전한 통신 가능
- 외부에서는 VPN 서버의 IP만 확인 가능
- 기업 원격 근무 환경 구축에 적합
- 지역 제한 우회 가능

------

## (-)

- VPN 제공 업체를 신뢰해야 함
- 서버를 한 번 더 거치므로 속도가 감소할 수 있음
- DNS Leak, IPv6 Leak 등의 설정 문제 발생 가능
- VPN 서버 장애 시 모든 통신이 중단될 수 있음
- 완전한 익명성을 제공하지는 않음
