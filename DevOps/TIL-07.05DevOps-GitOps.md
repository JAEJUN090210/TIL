# TIL - 07.05 / DevOps - GitOps

## Summary

- **GitOps는 Git 저장소를 단일 진실 공급원(Source of Truth)으로 사용하여 인프라와 애플리케이션을 선언적으로 관리하는 운영 방식**

------

## Body

- **GitOps**
  - Git에 저장된 설정을 기준으로 시스템을 운영하는 방식
  - Git의 변경 사항이 실제 운영 환경에 자동으로 반영
  - Kubernetes 환경에서 주로 사용
- **동작 과정**
  1. 개발자가 Git에 코드 또는 Kubernetes 매니페스트 변경
  2. 변경 사항이 저장소에 반영
  3. GitOps 도구가 변경 사항 감지
  4. 클러스터와 Git 상태 비교
  5. 차이가 있으면 자동 동기화(Sync)
- **주요 특징**
  - Git이 시스템의 **Source of Truth** 역할 수행
  - 선언형(Declarative) 방식으로 리소스 관리
  - Git 변경 이력이 운영 이력이 됨
  - 운영 환경과 Git 상태를 항상 일치하도록 유지
- **대표적인 GitOps 도구**
  - Argo CD
  - Flux CD

------

## (+)

- Git만으로 인프라 및 애플리케이션 관리 가능
- 변경 이력과 롤백이 쉬움
- 운영 환경의 일관성 유지
- 자동 동기화 및 Self Heal 지원
- 협업 및 코드 리뷰 기반 운영 가능

------

## (-)

- GitOps 개념과 Kubernetes에 대한 이해가 필요
- 잘못된 Git 변경 사항도 자동 반영될 수 있음
- 초기 구축과 권한 관리가 비교적 복잡함
- Kubernetes 환경에 가장 적합하며 다른 환경에서는 활용이 제한적일 수 있음
