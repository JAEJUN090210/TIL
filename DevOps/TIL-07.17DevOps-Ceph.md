# TIL - 07.17 / DevOps - Ceph

## Summary

- **Ceph는 분산 환경에서 블록(Block), 파일(File), 오브젝트(Object) 스토리지를 하나의 플랫폼으로 제공하는 오픈소스 분산 스토리지 시스템**

------

## Body

- **Ceph**
  - 여러 서버의 디스크를 하나의 **통합 스토리지**처럼 사용할 수 있도록 하는 오픈소스 분산 스토리지
  - 데이터 복제와 장애 복구를 자동으로 수행하여 **높은 가용성(High Availability)**과 **확장성(Scalability)** 제공
  - 일반 서버만으로도 대규모 스토리지 클러스터를 구성 가능
- **주요 구성 요소**
  - **OSD (Object Storage Daemon)**
    - 실제 데이터를 저장하는 프로세스
    - 데이터 복제, 장애 감지 및 복구 수행
    - 일반적으로 디스크 하나당 하나의 OSD를 구성
  - **MON (Monitor)**
    - Ceph 클러스터의 상태와 메타데이터를 관리
    - OSD의 상태와 클러스터 정보를 유지
    - 일반적으로 홀수 개(3개 또는 5개)로 구성하여 장애에 대비
  - **MGR (Manager)**
    - 클러스터의 성능 및 상태를 모니터링
    - Dashboard, Prometheus 등 다양한 관리 기능 제공
  - **MDS (Metadata Server)**
    - CephFS(File System)의 메타데이터를 관리
    - 파일 및 디렉터리 정보를 처리
    - CephFS를 사용할 때만 필요
  - **RADOS**
    - Ceph의 핵심 분산 스토리지 계층
    - 데이터 저장, 복제, 장애 복구를 담당
    - 모든 Ceph 서비스가 RADOS 위에서 동작
- **일반적인 Ceph 동작 흐름**
  1. 클라이언트가 데이터 저장 요청
  2. MON이 클러스터 정보를 제공
  3. 클라이언트가 CRUSH 알고리즘을 통해 저장 위치 계산
  4. 해당 OSD에 데이터를 저장
  5. Replica 또는 Erasure Coding 정책에 따라 데이터 복제
  6. 장애 발생 시 다른 OSD에서 자동 복구
  7. 클라이언트는 동일한 방식으로 데이터 조회

------

## (+)

- 여러 서버를 하나의 대용량 스토리지처럼 사용 가능
- 장애 발생 시 자동 복구(Self-Healing)
- 수평 확장(Scale-Out)이 쉬움
- Block, File, Object 스토리지를 모두 지원
- 일반 서버(HDD/SSD)만으로 구축 가능
- Kubernetes, OpenStack 등과 높은 호환성

------

## (-)

- 작은 규모에서는 오버헤드가 큼
- 네트워크 성능에 영향을 많이 받음
- 장애 복구 시 디스크와 네트워크 사용량이 증가할 수 있음
- 모니터링과 용량 관리가 필수적임
