# TIL - 07.22 / DevOps - Convolution

## Summary

- **Convolution(합성곱)은 입력 데이터에 작은 필터(Filter)를 적용하여 특징(Feature)을 추출하는 연산**

------

## Body

- **Convolution (합성곱)**
  - 입력 데이터 위를 작은 **필터(Filter 또는 Kernel)**가 이동하며 연산을 수행하는 기법
  - 이미지의 중요한 특징(모서리, 선, 패턴 등)을 추출하는 데 사용
  - CNN의 핵심 연산으로 이미지 인식, 객체 탐지 등에 활용
- **주요 구성 요소**
  - **Input**
    - 연산의 대상이 되는 입력 데이터
    - 일반적으로 이미지 또는 특징 맵(Feature Map)
  - **Filter (Kernel)**
    - 입력 데이터에 적용되는 작은 행렬
    - 특정 패턴을 검출하도록 학습됨
    - 대표적으로 3×3, 5×5 크기를 사용
- **Convolution의 특징**
  - **Feature Extraction**
    - 이미지의 경계, 모서리, 질감 등의 특징 추출
  - **Parameter Sharing**
    - 하나의 필터를 전체 이미지에 반복 사용
    - 파라미터 수를 크게 줄일 수 있음
  - **Local Connectivity**
    - 입력 전체가 아닌 작은 영역만 연산
    - 공간적인 특징을 효과적으로 학습

------

## (+)

- 이미지의 특징을 효과적으로 추출 가능
- 파라미터 수가 적어 학습 효율이 높음
- 위치 변화에 비교적 강인한 특징 학습 가능
- 다양한 크기의 필터를 사용할 수 있음
- 이미지 분류, 객체 탐지, 영상 분석 등에 널리 활용

------

## (-)

- 필터 크기와 개수에 따라 계산량이 증가할 수 있음
- 깊은 네트워크에서는 연산 비용이 커질 수 있음
- 복잡한 패턴 학습에는 많은 데이터가 필요
- 하이퍼파라미터(Filter, Stride, Padding) 설정이 성능에 큰 영향을 미침
- 순차 데이터 처리에는 Transformer 등 다른 구조가 더 적합한 경우도 있음
