# TIL - 06.28 / DevOps - CI/CD

## Summary

- **CI/CD는 소스 코드의 변경 사항을 자동으로 빌드, 테스트, 배포하는 개발 자동화 방식**

------

## Body

- **CI (Continuous Integration, 지속적 통합)**
  - 개발자가 코드를 저장소에 병합하면 자동으로 **빌드 및 테스트**를 수행
  - 오류를 빠르게 발견하고 품질을 유지
- **CD (Continuous Delivery / Continuous Deployment)**
  - **Continuous Delivery:** 배포 가능한 상태까지 자동화하고, 실제 배포는 사람이 승인
  - **Continuous Deployment:** 테스트를 통과하면 운영 환경까지 자동 배포
- **일반적인 CI/CD 흐름**
  1. 코드 작성
  2. Git 저장소에 Push
  3. CI 서버가 자동 빌드 및 테스트
  4. Docker 이미지 생성
  5. 이미지 레지스트리에 업로드
  6. Kubernetes 등 운영 환경에 배포

------

## (+)

- 반복 작업 자동화
- 빠른 배포와 피드백
- 사람의 실수 감소
- 코드 품질 및 안정성 향상
- 개발 생산성 향상

------

## (-)

- 초기 구축 비용과 시간이 필요
- 파이프라인 관리가 복잡할 수 있음
- 잘못된 설정 시 오류가 자동으로 배포될 위험 존재
- 테스트가 충분하지 않으면 자동 배포의 위험성이 커짐
