# TIL - 07.02 / Web - A11Y

## Summary

- **웹 접근성(A11Y)은 장애 유무와 관계없이 모든 사용자가 웹 서비스를 동등하게 이용할 수 있도록 하는 원칙과 기술**

------

## Body

- **A11Y (Accessibility)**
  - *Accessibility*의 약어 (`a`와 `y` 사이 11글자 생략)
  - 국내에서는 **KWCAG(한국형 웹 접근성 지침)**을 기준으로 적용
- **목적**
  - 정보 접근의 차별 제거
  - 모든 사용자가 동일한 사용자 경험 제공
  - 웹 서비스의 공공성과 사용성 향상
- **적용 대상**
  - 공공기관 및 교육기관(의무)
  - 민간 서비스(권장, 실무에서는 사실상 필수)
- **POUR 4대 원칙**
  - **Perceivable(인식 가능)** : 대체 텍스트, 자막 제공, 색상만으로 정보 전달 금지
  - **Operable(운용 가능)** : 키보드 조작 지원, 포커스 관리
  - **Understandable(이해 가능)** : 명확한 안내와 오류 메시지 제공
  - **Robust(견고성)** : 시맨틱 HTML 및 보조기기와의 호환성 확보

------

## (+)

- 장애인과 고령자도 서비스를 이용 가능
- 사용자 경험(UX) 향상
- 검색 엔진 최적화(SEO)에 도움
- 다양한 브라우저 및 보조기기와의 호환성 향상

------

## (-)

- 초기 개발 시 고려해야 할 요소가 많음
- 테스트 및 유지보수 비용 증가
- ARIA 속성 등을 잘못 사용하면 오히려 접근성이 저하될 수 있음
